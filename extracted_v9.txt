  TECHNICAL ARCHITECTURE
============================================================

  1. MODEL SELECTION
     Primary:  PaddleOCR v3 (PP-OCRv5, lang=ar)
       - Built-in DB (Differentiable Binarization) text detector
       - CRNN recognition head trained on Arabic script
       - Processes raw images (no pre-binarisation needed)
     Fallback: Tesseract LSTM (--oem 3 --psm 6 -l ara)
       - Receives pre-processed binary image (binarised + deskewed)
     Native:   PyMuPDF vector text for digital pages (no OCR needed)

  2. LAYOUT ANALYSIS
     Table detection: 3-layer hybrid pipeline (detect_tables_cv.py)
       Layer 1: Morphological line isolation (bordered tables)
       Layer 2: PyMuPDF word-band analysis (borderless digital tables)
       Layer 3: OpenCV adaptive threshold (scanned page fallback)
     Region classification: spatial_data.json bounding boxes
     Non-table regions: everything outside detected table boxes

  3. SPATIAL CORRELATION
     Method: Y-coordinate proximity matching
     Threshold: ±15 pt
     Text lines within threshold of a table row →
       tagged as TABLE_ANNOTATION with back-reference

  4. PRE-PROCESSING (for scanned pages)
     a) DPI normalisation — upscale to ≥300 DPI (INTER_CUBIC)
     b) CLAHE contrast enhancement (clipLimit=2.0, 8×8 grid)
     c) Gaussian blur (5×5) — scanner noise reduction
     d) Adaptive Gaussian binarisation (blockSize=31, C=10)
     e) Morphological closing (2×2) — repair Arabic ligatures
     f) Deskew via minAreaRect angle detection (±10° range)

  5. RTL / BiDi LOGIC
     Library: python-bidi (✓ loaded)
     Span ordering: RTL (rightmost first per visual row)
     Column ordering: RTL (col 0 = rightmost = labels)
     Mixed content: Arabic text + LTR numbers separated
       structurally by column detection, not character reordering
     Output: logical order with U+200F (RLM) for correct rendering
